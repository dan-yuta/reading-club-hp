# 目的
オンライン読書会のLP（1ページ中心）を作る。
ゴールは「初心者が気軽に一歩踏み出して参加申込みする」こと。
不安の主要因は「意識高い人ばかりで怖そう」「知り合いがいなくて浮きそう」。
空気感は「フラットで仲間感が強い」＋「成長意欲が高い刺激的」。

# 技術要件
- GitHubで管理しNetlifyでデプロイする静的サイト
- フレームワークは使わず、HTML/CSS/JSの最小構成でよい（運用と保守が楽なため）
- ただし今後ページ追加やSEO強化がしやすいように構造化する
- 画像は後から差し替え・追加できるように `assets/images/` を用意
- コンテンツ（文章）は `content/` に分離し、HTML側から読み込む方式にする（更新が楽）
  - 例：JSON（推奨）またはMarkdownを採用
  - JSで `content/site.json` を読み込み、DOMに流し込む実装を優先（運用が楽）
- ルーティングは不要。基本は1ページ `index.html` で完結
- レスポンシブ対応（スマホ優先）
- アクセシビリティ（最低限：見出し構造、ボタンのfocus、altテキスト）を守る
- SEO（最低限：title/description/OGP、見出し設計、canonical、sitemapとrobotsは簡易でOK）
- 申込み導線は固定：Googleフォームに遷移する（下記URL）

# ページコンセプト
「一人参加が当たり前。安心して成長できる読書会」
- 威圧感を出さない（実績アピールや過度な煽りは避ける）
- ただし成長価値（アウトプット・視点拡張・継続）を明確にする
- 不安解消を最優先にし、安心情報を上部で提示する

# 追加要件（開催ルール / 本のルール：必ずページに明記）
- 開催は「土日祝日の朝8:00」開始で実施していること（開催頻度は後から変更できるようフィールド化）
- 持ち寄る本は「1冊のみ」でお願いしたいこと
  - 理由：2冊以上持参されても、発表・ディスカッションの時間内に収まりにくいため
- 扱う本のジャンル制限：
  - 専門書はNG（例：高度な技術専門書、学術書、資格試験の特定分野に偏る本など）
  - 小説は原則NG
  - ただし小説であっても「ビジネス書／自己啓発書に相当する内容（学び・行動変容・思考法など）が中心」であればOK
  - 基本は「ビジネス書」または「自己啓発書」を対象とする
- 「ビジネス書」「自己啓発書」の定義をページ内に簡潔に記載する（後から編集しやすいようコンテンツ化）
  - ビジネス書（定義案）：仕事・キャリア・組織・マネジメント・マーケティング・営業・戦略・お金/投資・起業など、仕事に関わる意思決定やスキル向上に役立つ内容を主題とする本
  - 自己啓発書（定義案）：習慣化・思考法・メンタル・学習法・時間管理・コミュニケーション・目標設定など、個人の行動変容や自己成長に役立つ内容を主題とする本
  - OK例/NG例を2〜3個ずつ用意（任意で増やせる構造）
- 上記は「厳格な審査」ではなく「参加者全員が話しやすく、学びが揃うためのガイドライン」というトーンで記載する

# 情報設計（セクション構成）
1. Hero（最上部）
   - キャッチ：一人参加が当たり前のオンライン読書会
   - サブ：初心者歓迎 / フラットな仲間 / アウトプットで成長
   - CTA：参加申込み（フォームURLは content から差し込めるように）
2. ようこそ（導入セクション：短文）
   - 「ようこそ！こちらは『シン・ビジネス、自己啓発読書会』の紹介ページです」
   - 「20〜30代で読書仲間が欲しい方に向けた、気軽に参加できるオンライン読書会」
   - ※見出し・本文は絵文字を使わず、上品なトーンで表現する
3. こんな方におすすめ
   - 文章を読む習慣を身につけたい
   - 読んだ本がなかなか定着しない
   - 読書仲間を見つけたい
   - 本をアウトプットしてみたい
4. 開催概要（上部にも短く出す / 詳細は後段）
   - 土日祝日の朝8:00開始（明記）
   - Zoomオンライン
   - 参加費無料
   - CTA（申込み）
5. 安心ポイント（3つ）
   - 初心者歓迎
   - ほぼ全員一人参加（数字は後から入れられるようにフィールドを用意。未入力なら非表示）
   - 発言は自由（聞くだけOKなど）
6. 成長できる理由（3つ）
   - 読む→言語化（アウトプット）
   - 他人の視点で視野が広がる
   - 継続できる仲間ができる
7. イベントの流れ（図解風）
   - 受付
   - 自己紹介
   - 本の発表/ディスカッション
   - 締め
   ※少人数なら短縮の注記
8. 本のルール（重要：独立セクション）
   - 本は1冊のみ（理由も併記）
   - ジャンル：ビジネス書／自己啓発書
   - 専門書NG、小説は原則NG（例外条件を明記）
   - ビジネス書・自己啓発書の定義（短く）
   - OK例/NG例
9. ルール（禁止事項）
   - ネットワークビジネス・宗教の勧誘、営業活動
   - 他のメンバーへの迷惑行為や誹謗中傷
   - 無断キャンセルの繰り返し
   - 「主催者も遵守する」旨
   - 体調不良時は回復優先の案内（丁寧に）
10. 基本情報（開催概要：詳細）
   - 対象者（20-30代など）
   - 持ち物（読み終えた本1冊、Kindle可）※1冊限定を再掲
   - 参加費（無料）
   - 場所（Zoom、基本顔出し）※ON/OFF可能なフィールドにする
   - 応募締切（前日23:59）
11. 申込みCTA（下部）
   - 「参加希望の方はこちら」→ 申込みフォーム
   - 再度ボタン（上部と同じURL）
   - モバイルでは下部固定CTA（任意）
12. フッター
   - 過去の紹介本（noteリンク）
   - 「今後noteで紹介本を記録していく」旨（上品に短く）
   - SNSや連絡先（任意）

# デザイン要件（トーン / UI）
- 「安っぽい絵文字」を使わない（ページ本文・見出し・ボタン含む）
- アイコンが必要な場合は、絵文字ではなくSVG（線画）を使う
- 白（#FFFFFF）と薄い水色を基調としたクリーンな配色にする
  - 背景：白〜ごく薄い水色のグラデーション（控えめ）
  - アクセント：薄い水色（例：#D9F1FF や #BFE6FF など、濃すぎない）
  - 文字：可読性重視で濃いグレー（例：#1F2937相当）
  - 罫線・カード境界：薄いグレー〜水色寄りの境界
- 余白と行間を重視し、上品で落ち着いた印象にする
- フラット＆仲間感：角丸、カードUI、柔らかいシャドウは控えめ
- 成長刺激：見出し・要点の強調でメリハリ（煽り表現はしない）
- 読みやすさ：1セクションの文章量を抑え、箇条書き中心
- CTAは1ページに複数箇所（上部と下部）
- 画像枠をあらかじめ用意（後から差し替えやすく）
  - Hero背景画像（任意）
  - 雰囲気画像（任意）
  - 図解画像（任意）

# content/site.json の仕様（例）
- site.title
- site.description
- links.apply_url（参加申込みURL）※上記GoogleフォームURLを初期値として設定
- links.note_url（過去の本）
- hero.headline / hero.subheadline / hero.badges[]（バッジは文字のみ。絵文字禁止）
- intro.title / intro.body（「ようこそ」導入文）
- recommended.items[]（「こんな方におすすめ」）
- schedule.summary（例：土日祝 8:00開始）
- reassurance.items[]（安心ポイント）
- growth.items[]（成長ポイント）
- flow.steps[]
- book_rules
  - one_book_only（true）
  - one_book_reason（理由文）
  - allowed_genres[]（ビジネス書/自己啓発書）
  - disallowed_genres[]（専門書/小説など）
  - fiction_exception_note（小説OKの例外説明）
  - definitions.business（ビジネス書の定義）
  - definitions.self_help（自己啓発書の定義）
  - examples.ok[] / examples.ng[]（OK例/NG例）
- faq.items[]（QとA）
- basics（対象者/持ち物/参加費/場所/締切/顔出しポリシーなど）
- rules.prohibited[]（禁止事項）
- rules.notes[]（主催者も遵守、体調不良時の案内など）
- footer.message（noteで記録していく旨の上品な文章）
- images.hero（任意、未設定なら表示しない）
- images.flow（任意、未設定なら表示しない）
- images.community（任意、未設定なら表示しない）

# 実装要件（JS）
- main.js で `content/site.json` を fetch してページを組み立てる
- JSONの値が空・未設定の場合は該当ブロックを非表示にする（運用を楽に）
- CTAボタン（上部・下部）は links.apply_url（Googleフォーム）にリンク
- FAQはアコーディオン（details/summaryでOK、JS不要でも可）
- ヘッダー固定は不要。ただしモバイルでCTAが迷子にならない工夫（下部固定CTAは任意）

# 実装要件（CSS）
- 余白と行間重視
- 見出し階層を守る（h1→h2→h3）
- カードUIで「おすすめ」「安心」「成長」「本のルール」「基本情報」を整理
- ボタン・カード・罫線は白＋薄い水色の上品なトーンで統一
- できればダークモード対応（任意）

# 画像運用ルール
- assets/images/ に置き、content/site.json から相対パスで参照
- 画像を差し替えるだけで表示が変わるようにする
- READMEに推奨サイズを記載（例：hero 1600x900、flow 1200px幅など）

# 納品物
- 上記のフォルダ・ファイルをすべて作成
- content/site.json は日本語で仮文面を入れて完成させる（あとで差し替え可能）
  - 導入文（ようこそ）
  - おすすめ対象
  - 開催（土日祝8:00）情報
  - 本のルール（1冊のみ＋理由、ジャンル条件、定義、例）
  - ルール（禁止事項）
  - 流れ
  - 開催概要
  - フッターメッセージ
  を必ず含める
- READMEに「更新は基本 site.json を触るだけ」と明記し、手順を箇条書きで説明
- エラーなくブラウザで表示できる状態にする

# 仮文面（site.jsonに入れる内容の方向性）
- 口調は丁寧で落ち着いたトーン。絵文字は使わない
- 「一人参加歓迎」「初心者歓迎」「聞くだけOK」「発言強制なし」を明確に書く
- 「アウトプットで定着」「他の人の視点」「継続できる仲間」で成長価値を示す
- 本のルールは「場の学びを揃えるためのガイドライン」という柔らかいトーンで説明する
- 禁止事項（勧誘等）と、体調不良時の参加見送り案内は丁寧に記載
- 「今後noteで紹介本を記録していく」旨はフッターに短く上品に記載
- CTA文言は「参加申込み」または「お申し込みフォームへ」

# 追加（SEO最低限）
- index.htmlに meta title/description/OG（仮でOK）
- canonicalはNetlify本番URLを後から設定しやすいようコメントでガイドを入れる
- public/robots.txt と public/sitemap.xml を用意（1ページでOK）

以上を満たす形で実装してください。
